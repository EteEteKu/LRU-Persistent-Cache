# ⚡ LRU Persistent Cache

[![C++](https://img.shields.io/badge/C++-17-blue.svg)](https://en.cppreference.com/w/cpp/17)
[![CMake](https://img.shields.io/badge/CMake-3.15+-brightgreen.svg)](https://cmake.org/)
[![Tests](https://img.shields.io/badge/Tests-Catch2-important.svg)](https://github.com/catchorg/Catch2)

**LRU Persistent Cache** — это легковесное консольное key-value хранилище с фиксированным размером кэша, реализующее алгоритм вытеснения **Least Recently Used (LRU)**. Ключевая особенность — возможность сохранять состояние кэша на диск (создавать снапшоты) и восстанавливать его при следующем запуске.

## 🚀 Основные возможности

- **LRU-кэш**: Эффективное хранение данных с автоматическим вытеснением наименее востребованных элементов при превышении лимита.
- **Персистентность**: Сохранение текущего состояния кэша в файл-снапшот и его загрузка при инициализации.
- **Консольный интерфейс**: Взаимодействие через команды `SET`, `GET` и `DEL`.
- **Атомарное сохранение**: Запись снапшота во временный файл с последующим переименованием для предотвращения повреждения данных.
- **Модульные тесты**: Покрытие ключевой логики тестами на базе фреймворка **Catch2**.

## 🛠️ Сборка и запуск

### Требования

- Компилятор с поддержкой C++17
- [CMake](https://cmake.org/) (3.15 или выше)

### Инструкция по сборке

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/EteEteKu/LRU-Persistent-Cache.git
   cd LRU-Persistent-Cache/LRU-Persistent-Cache
   ```

2. Создайте директорию для сборки:
   ```bash
   mkdir build && cd build
   ```

3. Сгенерируйте файлы сборки:
   ```bash
   cmake ..
   ```

4. Выполните сборку:
   ```bash
   cmake --build .
   ```

   Исполняемый файл будет создан в папке `build`.

### Использование

Запустите собранный исполняемый файл. Программа загрузит снапшот из файла `snapshot.dat` (если он существует) и начнет принимать команды.

**Доступные команды:**

| Команда | Описание |
| :--- | :--- |
| `SET <ключ> <значение>` | Добавляет или обновляет пару ключ-значение. |
| `GET <ключ>` | Получает значение по ключу. Если ключ не найден, выводится сообщение. |
| `DEL <ключ>` | Удаляет пару ключ-значение. |
| `exit` | Завершает работу программы. |

**Пример сессии:**
```text
SET name Alice
Значение успешно set
GET name
Alice
DEL name
Объект удален успешно
exit
```

## 🧩 Архитектура

Проект построен на двух основных классах:

1. **`LRUcache`**: Реализует алгоритм LRU.
   - Использует `std::list` для хранения пар (ключ, значение) в порядке использования.
   - Использует `std::unordered_map` для быстрого доступа к элементам списка по ключу.
   - Предоставляет методы `put()`, `get()`, `del()`.

2. **`KVstore`**: Обёртка над `LRUcache`, добавляющая персистентность.
   - Делегирует операции с данными экземпляру `LRUcache`.
   - Управляет созданием снапшотов (`saveSnapshot()`) и их загрузкой (`loadSnapshot()`).

## 🧪 Тестирование

Для запуска тестов выполните в директории `build`:

```bash
ctest
```

Тесты, написанные с использованием Catch2, покрывают:
- Базовые операции `KVstore`.
- Логику вытеснения и обновления ключей в `LRUcache`.
- Корректность сохранения и загрузки снапшотов.
